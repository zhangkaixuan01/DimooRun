# Evaluator Path

Goal: decide within 15 minutes whether DimooRun fits a team that already has
agent code and needs runtime control.

## Path

1. Read [Product Overview](product-overview.md) for boundaries and non-goals.
2. Run [Quickstart](quickstart.md) with `examples/langgraph/support-agent`.
3. Open the Console Dashboard and follow Publish, Deploy, Run, Inspect.
4. Open the run detail page and verify version, deployment, input/output,
   events, artifacts, and audit links where present.
5. Check [Current Maturity](../readiness/current-maturity.md) before making any
   production-readiness claim.

## Done

- You can explain why DimooRun is a runtime control plane, not an agent
  framework or low-code builder.
- You have a real `run_id` created through the runtime API.
- You have inspected the run evidence in Console or through CLI output.
