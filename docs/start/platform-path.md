# Platform Path

Goal: understand whether DimooRun can fit into platform engineering operations.

## Path

1. Run the Compose quickstart and record local proof.
2. Review runtime, worker, capacity, scheduled/batch, cost, and observability
   pages.
3. Review settings for provider status, exporters, sandbox policies, platform
   defaults, backup/restore, and danger zone actions.
4. Review Helm, Compose, readiness scorecard, and operations runbook before
   making deployment claims.
5. Track remaining gaps in [Readiness Gap Backlog](../readiness/gap-backlog.md).

## Done

- You can explain the control plane, runtime plane, and worker split.
- You can identify which local proofs exist and which hosted proofs are missing.
- You know the next hardening work before production-grade claims.
