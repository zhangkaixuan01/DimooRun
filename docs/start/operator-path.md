# Operator Path

Goal: operate deployed agents through evidence-first workflows.

## Path

1. Open Dashboard and review next actions, queue pressure, worker health, recent
   failures, and pending approvals.
2. Open Deployments to inspect active version, desired status, runtime status,
   and rollback context.
3. Open Runs to inspect failed or recent runs.
4. Use run detail to inspect attempts, events, input/output, artifacts, trace
   ids, and audit evidence.
5. Use replay, approval, rollback, or incident workflows when the evidence
   points to those actions.

## Done

- You can identify the next operational action from Dashboard.
- You can answer which deployment and AgentVersion handled a run.
- You can find the evidence needed for replay, approval, rollback, or audit.
