# Roadmap

This roadmap describes direction, not a release promise. It should stay aligned
with the production-grade gap-closure plan and remain conservative about timing.

## Principles

- prefer executable proof over optimistic claims
- ship product workflows, not just route coverage
- keep framework compatibility without hiding native governance boundaries
- treat operations, audit, and evidence as product features

## Now

Current focus areas:

- hosted Compose and Kubernetes smoke proof
- broader browser workflow graduation from partial to complete
- stronger trust, example, and community assets
- release and artifact publication proof

## Next

Expected next hardening layers:

- more complete trust and security verification
- generated screenshots and demo evidence
- stronger product onboarding and contributor workflows
- deeper hosted runtime and deployment proof

## Later

Longer-range product areas already visible in the plan:

- richer cost and budget operations
- scheduled and batch workflows
- catalog and asset workflows
- broader enterprise operations and competitive product polish

## Boundaries

This roadmap is not a promise of:

- multi-region or hyperscale guarantees
- vendor-neutral parity across every external framework surface
- every optional enterprise workflow becoming part of the near-term baseline
